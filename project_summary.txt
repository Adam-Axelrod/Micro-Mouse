### **Project: Autonomous Micro-Mouse Agent with Deep Reinforcement Learning**

**Overview:**
Developed an autonomous agent capable of solving complex mazes using **Deep Reinforcement Learning (Deep Q-Learning / DQN)**. The system uses a custom-built simulation environment (Pygame) where the agent learns optimal pathfinding strategies through trial and error, moving beyond simple heuristic algorithms like Wall Following or Flood Fill.

**Key Technical Implementation (PyTorch & Data Science):**

*   **Deep Q-Network (DQN) Architecture:**
    *   Designed and implemented a feedforward neural network (MLP) using `torch.nn.Module` with 3 linear layers and ReLU activation.
    *   **Input Layer (16 neurons):** Engineered a custom state space combining **simulated lidar/ray-casting** (8 directions for wall distance sensing), relative waypoint coordinates (path planning), and agent kinematics (normalized angle/speed).
    *   **Output Layer (4 neurons):** Predicts Q-values for discrete actions (Move Forward/Backward, Turn Left/Right).
    *   **Optimization:** Utilized `torch.optim.Adam` with Mean Squared Error (MSE) loss to minimize the temporal difference error between predicted Q-values and target Q-values (Bellman equation).

*   **Reinforcement Learning Pipeline:**
    *   **Experience Replay:** Implemented a custom replay buffer (`deque`) to store transitions `(state, action, reward, next_state, done)`.
    *   **Prioritized Sampling:** Engineered a sophisticated sampling strategy that weights experiences based on **reward magnitude and recency**, improving learning efficiency for rare, high-value events.
    *   **Reward Shaping:** Designed a complex reward function to guide the agent, including:
        *   Dense rewards for progress towards waypoints ("reward gates").
        *   Exponential distance penalties for deviating from the optimal path.
        *   Sparse rewards for reaching the goal.
    *   **Training Loop:** Built a custom training loop handling the exploration-exploitation trade-off (Epsilon-Greedy with decay), tensor batch processing, and gradient clipping to prevent exploding gradients.

**Key Skills Demonstrated:**
*   **PyTorch:** Custom model architecture, tensor manipulation, gradient descent, model saving/loading.
*   **RL Fundamentals:** Q-Learning, Policy optimization, Reward Engineering, State Space Design.
*   **Data Handling:** Real-time data preprocessing (normalization of sensor inputs), efficient memory management for experience replay.

This highlights your ability to not just use pre-built models, but to **architect custom neural networks**, **engineer features (state space)**, and **implement core RL algorithms from scratch** in PyTorch.
